Media Queries and Accessibility

This week we learned about how to use media queries in our CSS in order to change the styling of our HTML for different screen sizes. This allows us to make our mobile, tablet, and desktop versions of our sites to all still look great despite their size differences.

You can set sizes using units like px, em, rem, vw/vh, or percentages. Pixels are absolute lengths that don't adapt to different screen sizes. Em and rem are relative units in that their size will adjust based on the size of their parent (em) or root (rem) element font size. Percentages are also relative units and set the size relative to the parent element. View width (vw) and view height (vh) are completely reliant on the width and heigh of the viewport.

We also learned about different site layout types: fixed, fluid, adaptive, and responsive.

Fixed sites are designed with one layout styling in mind (usually desktop), so it is not "friendly" for other screen sizes because the content doesn't adjust.

Fluid sites adapt to different screen sizes, but they grow and shrink proportionally. This can make the site look weird because things like fonts and images can get too small or big.

Adaptive sites use media queries to change the styling on a site for different specified screen sizes. This can help avoid the funky proportions that can be caused by fluid sites. However, widths are still hard coded.

Finally, responsive sites also use media queries to adapt to different screen sizes. However, they are different from adaptive sites in that they still use responsive units for widths and other sizings instead of hard coded sizes so that the site can always adapt to the screen sizes that don't have their own media queries.

Thoughts and Experience

I really enjoyed the lessons and projects this week! I loved that in class they really emphasized the importance of adaptive and responsive websites, not just for better user experience, but to make websites more accessible. For example, hard coding things like font sizes can make things difficult for people with visual impairments that rely on accessibility zoom features in browsers because the font won't adjust as needed. It's important that when we structure our HTML and design our CSS that we keep accessibility in mind. No one should be excluded from being able to access sites simply because of the way a site was designed.
